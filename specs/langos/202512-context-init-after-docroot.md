# 在 doc_root 选定后自动初始化上下文的规格

## 背景与目标
- 问题：当前在选择 doc_root 后，AI 仍可能重复询问“这是个什么项目”“有哪些 repo”等基础信息。
- 目标：doc_root 选定后，运行时自动做一次轻量的上下文初始化，获取项目定位与 repo 清单，避免后续重复问已在文档目录说明的内容。

## 触发与范围
- 触发时机：doc_root 确认/切换成功后立即执行（启动流程或中途切换均生效）。
- 作用范围：当前会话的主动记忆，不写入业务文档。
- 目标仓库：默认用于 langos 内核场景；若切换 doc_root 指向其他项目文档仓，沿用相同行为。

## 读取范围（最小集合）
- 必读：doc_root 仓根 README、`repos/INDEX.md`、与 active repo 对应的 overview（例如 `repos/langos.md`）。
- 可读（若存在）：`blueprints/vision.md`、`blueprints/meta-intro.md`、`meta/README.md`、相关 specs 目录的入口/README（仅用于了解项目定位与分类方式）。
- 不读：未明确的敏感目录或大体量文档；无须全文解析，只需建立心智。

## 行为与输出
- 行为：静默读取上述文件并建立心智模型（项目是什么、有哪些 repo、目录职责等）。
- 输出：无需生成长摘要，最多一句确认（例如“已获取项目概览与 repo 列表”）。
- 后续对话：不再询问文档目录中已说明的基础信息（例如项目定位、repo 清单）；若信息缺失则按缺项提示。

## 安全与边界
- 缺文件时提示缺项，不视为错误。
- 不缓存或暴露文档全文；仅在会话中使用。
- 保持中文对话，引用路径/文件名保持原文。

## 验收标准
- doc_root 选定后自动执行上述读取；若 doc_root 缺必要文件，明确提示缺项。
- 后续对话不再重复追问“这是个什么项目”“有哪些 repo”等已覆盖信息。
